## 一、研究定位

### 核心命題

> 當 AI 能力持續提升、成本持續下降，人類勞動市場會如何重組——以及什麼特質讓人在這個過程中存活？

這個命題涵蓋三個層次：

- **結構問題**：產業與職業的分佈如何重組
- **動態問題**：AI vs Human 曲線如何交錯演化
- **個體問題**：什麼特質在這個過程中有存活優勢

### 研究方法：Agent-Based Modeling（ABM）

選擇 ABM 而非統計預測模型的理由：

- 不做「經濟預測」，做「結構演化」
- 讓市場動態從 Agent 行為中**自然浮現**，而非預設結果
- 可以在受控環境下跑政策實驗（UBI、再教育補貼等）

### 比賽定位建議

| 評審在意的 | 對應策略 |
|---|---|
| 問題夠不夠真實、重要 | 明確的現實對應，聚焦三個研究問題 |
| 方法有沒有獨特性 | ABM 本身是亮點，需說明為何優於統計方法 |
| 結果有沒有洞見 | 要有反直覺或細緻的發現，不能只是「AI 會取代人」 |

## 二、職業分類框架：四維度

### 問題意識

原始的職業分類（製造業/金融/創意/客服）無法涵蓋水電工、美髮師等「實體服務業」，因為這類職業的關鍵限制不是技能深度，而是任務的物理性與社會嵌入性。

### 四個維度定義

#### 維度一：數位化程度（Digitizability）

**核心問題**：任務能否轉為符號/資訊處理？

| 低端 | 高端 |
|---|---|
| 需實體在場、觸覺、空間感 | 可遠端執行、輸入輸出皆為數據 |
| 水電工、美髮師 | 程式設計、資料分析 |

**學術根源**：Autor, Levy & Murnane（2003）的 cognitive vs manual 分類；Frey & Osborne（2017）的 perception & manipulation 瓶頸。

---

#### 維度二：情境依附性（Contextual Embeddedness）

**核心問題**：任務是否嵌在特定社會關係或物理場域中？

| 低端 | 高端 |
|---|---|
| 可標準化、脫離脈絡執行 | 需要信任、情感連結、實體存在 |
| 資料輸入、基礎行政 | 護理師、心理諮商、美髮師 |

**學術根源**：Frey & Osborne 的 social intelligence 瓶頸；Deming（2017）關於社交技能在勞動市場的持久價值。

---

#### 維度三：認知深度（Cognitive Depth）

**核心問題**：任務需要多深的專業判斷？

| 低端 | 高端 |
|---|---|
| 規則明確、有標準答案 | 需要診斷、推理、跨域整合 |
| 行政文書、基礎客服 | 外科醫師、策略顧問 |

**學術根源**：Acemoglu & Autor（2011）的 non-routine cognitive 分類；Frey & Osborne 的 creative intelligence 瓶頸。

---

#### 維度四：結構化稀缺性（Structuring Scarcity）

**核心問題**：任務中有多少比例，是在將「尚未被定義的輸入」轉換成「可以被執行的輸出」？

| 低端 | 高端 |
|---|---|
| 輸入已結構化，只需執行 | 輸入是混亂、矛盾、隱性的，需先賦予形狀 |
| Junior 工程師寫 function | 架構師決定要建什麼系統 |
| 報稅、標準法律文件 | 醫師診斷、主廚設計菜單、社工評估個案 |

**跨領域泛化**：「把混亂轉為可操作」的能力存在於各領域：
- 軟體架構師：把模糊需求轉成可執行結構
- 主治醫師：把混亂症狀轉成可處理問題
- 主廚（資深）：把食材限制和客人期待轉成具體料理
- 社工：把複雜家庭處境轉成可介入的切入點
- 美髮師（資深）：把「我想要好看」轉成具體剪法

**學術根源（三個傳統）**：

| 學術傳統 | 關鍵詞 | 核心論點 |
|---|---|---|
| 勞動經濟學 | **Polanyi's Paradox**（Autor 2014） | 人類有隱性知識，無法被明確編碼，因此 AI 難以複製 |
| 認知科學 | **Ill-structured / Wicked Problems**（Rittel & Webber 1973） | 問題本身的結構程度，決定 AI 能否介入 |
| 管理學實驗 | **Jagged Frontier / Task Ambiguity**（Dell'Acqua et al. 2023） | 模糊性是比複雜性更好的 AI 替代性預測指標 |

> **關鍵發現（Dell'Acqua et al. 2023，Harvard/BCG 實驗）**：任務的模糊性（ambiguity）比複雜性（complexity）更能預測 AI 的效果——結構化任務獲益最穩定，模糊性任務則結果高度不確定。

### 重要的動態性質

第四維度有一個其他三個維度沒有的特殊性質：**它會隨 AI 能力動態移動**。

今天的「定義問題」，明天可能變成「執行問題」。AI 每提升一個能力門檻，就有一批「問題定義」任務被降格為「執行」任務。

這意味著模型裡這個維度應該是**動態的**，產生的核心張力是：

> 人類永遠在逃，但逃的速度是否快過 AI 擴張的速度？

### 四維度座標範例

| 職業 | 數位化程度 | 情境依附性 | 認知深度 | 結構化稀缺性 |
|---|---|---|---|---|
| 水電工 | 極低 | 高 | 中 | 中（現場診斷） |
| 美髮師 | 極低 | 極高 | 低中 | 中低 |
| 基礎客服 | 高 | 中→低 | 低 | 低 |
| Junior 工程師 | 極高 | 低 | 中 | 低 |
| 軟體架構師 | 極高 | 低 | 高 | 極高 |
| 外科醫師 | 中 | 極高 | 極高 | 高 |
| 策略顧問 | 高 | 中 | 極高 | 極高 |
| 行政文書 | 高 | 低 | 低 | 極低 |

## 三、模型設計

### 世界架構

```
World
├── Agents
│   ├── Workers（人類）
│   ├── AI Workers
│   └── Companies（企業）
├── Task Market（任務市場）
├── Market Matcher（匹配機制）
├── Learning System（學習系統）
├── Replacement Engine（替代引擎）
└── Statistics Tracker（統計追蹤）
```

### 以 Task Market 為中心的運作邏輯

每個 timestep 的流程：

```
1. Task Market 產生新任務（需求側）
2. Company 發包任務
3. Human / AI 競標任務（供給側）
4. 執行 → 產出薪資 / 收益
5. 學習 / 替代 發生
6. 統計更新
```

> **設計原則**：市場動態是自然浮現的，不是硬設定的。

### 時間尺度

| Timestep | 現實對應 |
|---|---|
| 1 step | 1 個月 |
| 12 steps | 1 年 |
| 模擬長度 | 10–20 年（120–240 steps） |

### Agent 設計

#### Worker（人類）

```python
@dataclass
class Worker:
    skills: np.ndarray        # shape (5,)，對應五種技能
    adaptability: float       # 學習速度
    wage_expectation: float
    status: str               # employed / retraining / unemployed
    retrain_timer: int        # 還需幾步完成再訓練
    breadth: float            # 技能廣度（通才 vs 專才）
    structuring_ability: float  # 結構化稀缺性對應的個體能力
```

#### AI Worker

```python
@dataclass
class AIWorker:
    capability: float         # 當前能力值（隨時間成長）
    cost: float               # 使用成本（隨時間下降）
    reliability: float        # 穩定性
    scope: List[str]          # 可處理任務類型
    maturity_by_task: Dict    # 各任務類型的成熟度（S曲線）
```

#### Company

```python
@dataclass
class Company:
    budget: float
    task_demand: Dict         # 各類任務的需求量
    hiring_policy: float      # 人 vs AI 比例偏好
    efficiency_pressure: float
```

### Task 設計

```python
@dataclass
class Task:
    skill_requirement: np.ndarray
    complexity: float
    value: float
    deadline: int
    # 四維度分數
    digitizability: float           # 數位化程度
    contextual_embeddedness: float  # 情境依附性
    cognitive_depth: float          # 認知深度
    structuring_scarcity: float     # 結構化稀缺性
    # 衍生的替代性
    substitutability: float         # = digitizability × (1 - contextual_embeddedness)
```

### 匹配與替代機制

**公司決策**：

```python
# 基礎分數
score_ai    = (ai_capability - error_penalty) / ai_cost
score_human = (human_skill_match) / human_wage

# 但加入四維度修正
ai_advantage = score_ai * task.substitutability
human_advantage = score_human * (
    task.contextual_embeddedness +
    task.structuring_scarcity * (1 - ai_maturity)
)

assign_to = "AI" if ai_advantage > human_advantage else "Human"
```

**替代概率**：

```python
P(replace) = sigmoid(ai_advantage - human_advantage)

# 延遲效應（合約、慣性、法規、信任）
effective_replacement = P(replace) * delay_factor(τ)
```

**技能退化與遷移**：

```python
if job_lost:
    skill_decay += time_unemployed
    retrain_cost = f(task.cognitive_depth, task.structuring_scarcity)
    # 結構化稀缺性高的任務，再訓練成本更高但價值也更高
```

## 四、研究假設與情境設計

### 三個研究問題

1. **產業結構演化**：不同四維度組合的職業，在 AI 滲透過程中如何分化？
2. **AI 能力/成本曲線 × 人類反應**：三條曲線（AI 能力↑、AI 成本↓、人類學習速度）交錯時，市場如何重組？
3. **個體特質的存活分析**：哪種特質組合的 Worker，在不同情境下薪資/就業率最高？

### 三種情境對比

| 情境 | AI 成長速度 | 人類學習速度 | 預期結果 |
|---|---|---|---|
| 慢替代 | 低 | 中 | 平穩過渡 |
| 快替代 | 高 | 低 | 結構性失業 |
| 協作型 | 高 | 高 | 新職業出現 |

### 產業代表性選擇（建議四類）

| 產業 | 數位化 | 情境依附 | 認知深度 | 結構化稀缺 |
|---|---|---|---|---|
| 實體服務業（水電/美髮） | 極低 | 極高 | 低中 | 中低 |
| 知識密集服務（法律/財務） | 高 | 中 | 高 | 中高 |
| 創意產業（設計/廣告） | 高 | 中 | 中高 | 高 |
| 軟體開發 | 極高 | 低 | 高 | 高（但動態下降） |

## 五、開發 Roadmap

### Phase 0：先定義假設（開始寫 code 前）

花一個小時寫下你真正想驗證的命題，例如：

- **假設 A**：AI 成本下降速度 > 人類學習速度時，中間技能（高數位化、低結構化稀缺性）最先崩潰
- **假設 B**：高「結構化稀缺性」的 Worker，在 AI 快速成長情境下薪資反而上升
- **假設 C**：再教育補貼延遲失業波，但不改變最終的結構分佈

### Phase 1：骨架（純規則，約 2 週）

- 5 種任務類型，各有 `ai_maturity` 參數（0→1，S 曲線成長）
- Worker 只有基礎屬性
- Company 只做一件事：比較 score 然後選擇

**目標輸出**：就一張圖——`employment_rate over time`，看是否自然出現替代曲線。

### Phase 2：加入人的複雜性（約 2 週）

- 四維度技能向量
- 學習行為（被替代 → 進入再訓練狀態）
- 薪資動態（供需影響）

**新輸出**：技能別薪資曲線、Gini 係數

### Phase 3：加入 AI 演化（約 1 週）

- AI Capability 用 S-curve 成長（不同任務成熟速度不同）
- AI Cost 指數衰減
- **關鍵**：`structuring_scarcity` 的 AI 前沿動態移動

**這時候「三條曲線交錯」才真的會出現。**

### Phase 4：政策實驗（可選）

每個政策做 A/B 對比，比較 Gini / 失業率 / 薪資分佈：

- **UBI**：每 step 給所有失業者固定收入
- **再教育補助**：降低 retrain_cost
- **AI 稅**：對使用 AI 的公司加收費用，補貼人力

## 六、預期觀察到的社會現象

### 1. 中間技能塌陷（Hollowing Out）

```
高數位化 + 低結構化稀缺性 → 最先消失（Junior 工程師、基礎客服）
高數位化 + 高結構化稀缺性 → 存活 + AI 協作放大產出（架構師、策略顧問）
低數位化 → 替代速度慢（水電工、美髮師）
```

### 2. 結構化稀缺性的反直覺現象

AI 越強，「把問題結構化」的人越值錢——因為 AI 能力的邊界（jagged frontier）需要人來判斷任務落在哪一側。

### 3. 過渡性失業波

```
AI adoption ↑ → 短期失業 ↑ → 再訓練後回升（但結構不同）
```

### 4. 新職業浮現

- AI workflow designer
- Prompt / system architect
- AI QA / evaluator
- 複雜情境的「問題框架師」

### 5. 技能通膨

技能價值下降速度 > 人類學習速度，導致持續的適應壓力。

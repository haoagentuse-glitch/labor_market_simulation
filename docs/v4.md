> 第一性原理重構。判準只有一條：**只留下「生成現象」所必需的東西。任何只是在描述的、或任何把結論預先設定進去的，都刪掉。**

## 0. 為什麼要做減法

v3 機制完整，但有兩處在偷渡結論、整體又太多旋鈕：

| v3 的問題 | v4 的處置 |
|---|---|
| O-ring 用「子任務全要對」把凸度**結構性設定**進去 | **刪除**。集中改由排序匹配浮現，可被推翻 |
| 四維度當引擎輸入 → 假精確、灌水參數 | **降級**為結果的描述標籤，不進引擎 |
| 8+ 旋鈕、校準靶太少 → 什麼都能生成 | 砍到 **約 4 個**，組成清楚的 2×2 相圖 |
| 三通道、p_true、worker 多特質、AI 多屬性 | 各自**收成單一純量**或由規則推導 |

核心信念：**一個能解釋一切的模型，等於什麼都沒解釋。** 先把骨架砍乾淨、跑得動、看得懂，再一次加一個回饋。

## 0.5 實作修正（v4.1）：宏觀賽跑是 r vs g，不是 r vs c

> 寫第一版程式時浮現的發現——正是最小模型該有的價值：它**否證**了 naive 的設計假設，並指出下一步該加什麼。

原本（§1、§2、§4 下文）把宏觀賽跑設成「前沿速度 `r` vs 任務生成速度 `c`」。實作後發現這是錯的：

- **靜態能力下，前沿 `F` 掃過固定的能力分佈，會讓所有 `a < F` 的人失業。**
- **任務生成 `c` 只在前沿「之上」加任務，而那些任務只有高能力者能接** → `c` 抬高頂端收入與集中度，卻**救不了被替代的中低能力者**。
- 因此純靜態核心**只會走向崩潰**，`c` 改變不了這個宏觀結局。

真正切換「崩潰 ↔ 重配置」的，是**人類適應速度 `g`（學習）與前沿速度 `r` 的賽跑**：

- `r ≫ g`：前沿跑贏學習 → 在崗者也被追上 → **崩潰**。
- `g ≳ r`：學習跟上 → 在崗者持續爬升、守住身位 → **重配置**。

所以：

- 程式已內建 `human_learning = g`，宏觀相圖改掃 **`r × g`**。
- `c` 降級為**分配側**的旋鈕（決定頂端機會與集中度），不再是宏觀體制的決定者。
- 這也把 v3 的「三條曲線」重新接了回來：真正的賽跑本來就是 **AI 前沿 vs 人類學習**。

下文 §1/§2/§4 保留原始推導以呈現思路，但凡提到「`r` vs `c` 決定崩潰／重配置」之處，**以本節為準**。

## 1. 不可化約的核心

### 三個純量（每個實體只用一個數）

| 實體 | 純量 | 意義 |
|---|---|---|
| 任務 | `σ` | 離 AI 前沿多遠＝需要多少「賦予結構」的能力 |
| 人 | `a` | 能力，異質分佈（尾巴形狀是可掃描參數，不寫死） |
| AI | `F(t)` | 前沿，隨時間以速度 `r` 上爬 |

> 為什麼任務只要一個數：對**動態**而言，v1 的四維度全部化約成「離前沿多遠」。四維是好的**描述層**，留作結果投影，但不餵進引擎。

### 一條規則（每步、每個任務）

```
若 σ < F − ε        → AI 獨力完成（替代），人不需要
若 σ ≥ F           → 需要 a ≥ σ 的人來賦予結構，人+AI 產出價值 V(σ)
                     合格的人裡，能力高的去做 σ 高的（排序匹配）
a 一直卡在 F 以下的人 → 沒有任何可做的任務 → 失業
```

ε 是前沿比較的雜訊（毛邊前沿，操作化 Dell'Acqua 的模糊性）——不需要另立 `p_true`，可替代性**就是** σ 對 F 的比較。

### 這條規則免費生成的東西（都不是設定的）

- **替代 vs 協作**：σ<F 與 σ≥F 兩個區，從一個門檻掉出來，不需要兩套機制。
- **存活門檻** = 當下的 F 本身。能力 a≥F 才有活路；「什麼特質存活」直接從前沿讀出。
- **逃跑賽跑** = F 上升速度 `r` vs 人類學習速度 `g` 的競爭（見 §0.5；原寫 `c`，實作後修正為 `g`）。
- **集中 vs 壓縮** = 由排序匹配 × 能力尾巴 × V(σ) 形狀浮現，**可測、可被推翻**。

## 2. 任務供給：唯一不能省的「加法」

純替代模型必然崩潰。要有非崩潰體制，就需要新任務持續產生。最小做法：

```
σ 從 [0, σ_max(t)] 抽樣，   σ_max(t) = σ_max0 + c·t
```

`c` = 邊緣新問題（更高 σ 的任務）冒出的速度，取代了 v3 §六.4 那段「新職業用講的」。

> **修正（見 §0.5）**：原以為 `c` 與 `r` 賽跑決定崩潰／重配置。實作後發現 `c` 只在前沿之上加任務，僅影響**頂端機會與集中度**，救不了被替代的中低能力者。宏觀體制由 **`r` vs 人類學習 `g`** 決定；`c` 是分配側旋鈕。

---

## 3. 匹配與報酬（最小、無偷渡）

合格的人（a ≥ F）裡，用**排序匹配（assortative）**：能力最高者配 σ 最高的任務。貪婪、O(n log n)：

```
任務依 σ 由大到小；worker 依 a 由大到小
指標掃過：能做就配（a ≥ σ），配到的人 earnings = V(σ)
最大的人拿最大的任務；做不動的任務沒人能做 → 留空
小能力者撿小任務；a < F 者全程配不到 → 失業
```

報酬 `V(σ) = σ^k`。`k` = 價值凸度（單一參數）。

> 關鍵：集中**不靠** O-ring 塞進來。它若出現，是「排序 + 重尾 + 凸 V」的後果。若調平尾巴或調 k=1 仍集中，才代表機制錯——這是 v3 做不到的可證偽性。

## 4. 活下來的參數 → 2×2 相圖

砍到底，自由參數剩約四個，組成兩個獨立的二維問題：

```
宏觀的賽跑：  前沿速度 r   vs   人類學習速度 g        → 崩潰 / 重配置   （見 §0.5 修正）
分配的結局：  能力尾巴 σ_a  ×   價值凸度 k            → 集中 / 壓縮
```

**主要交付物**：兩張相圖——`(r, g)` 平面標宏觀體制、`(σ_a, k)` 平面標分配結局。比 v3 的 8 維超立方體可信得多，也講得清楚。

最小參數表：

| 參數 | 意義 | 基線 |
|---|---|---|
| `r` | 前沿上升速度 | 0.003 / step |
| `g`（`human_learning`） | 人類學習速度（在崗者能力成長）→ **宏觀體制決定者** | 0（核心）/ 掃描 |
| `c` | 任務生成速度（σ_max 成長）→ **分配側**旋鈕，非宏觀決定者 | 0.003 / step |
| `ability_sigma` | 能力對數常態的 σ（尾巴重度） | 0.5 |
| `V_curvature k` | V(σ)=σ^k 的凸度 | 2.0 |
| `frontier_noise ε` | 前沿毛邊 | 0.03 |
| `steps` | 月 × 20 年 | 240 |

## 5.5 過程量與最小再訓練機制（v4.1 實作）

只有結果指標（就業率、Gini、集中度）無法解釋**因果過程**。v4.1 在不改變核心規則的前提下，補上兩類東西。

### 最小再訓練機制（第二個回饋的雛形）

原核心裡失業者（`a < F`）不會成長 → 再訓練成功率恆為 0。為了讓「再訓練」可被觀察，加入單一參數 `retrain_rate`（預設 0，不影響既有結果）：

```python
# 在崗者以 g 學習，失業者以 retrain_rate 再訓練
growth = np.where(employed, human_learning, retrain_rate)
a = a * (1 + growth)
```

這正是 §6 規劃的「人類適應」回饋的最小版本。

### 六個過程指標（兩張機制圖）

| 圖 | 指標 | 模型內定義 |
|---|---|---|
| 職業結構 | 各職業層級人口比例 | 把 σ 切成層級（低/中/高/精英）+ 失業，依 worker 接到的任務 σ 分類 |
| 職業結構 | AI 替代率 | σ<F 的任務「價值」佔比（σ^k 加權） |
| 職業結構 | 新職業生成率 | σ>σ_max0 的任務價值佔比（c 生成的高端新工作） |
| 技能演化 | 平均技能值 | 全體 / 在崗者的 mean(a)，疊前沿 F |
| 技能演化 | 再訓練成功率 | 上一步失業、這一步就業的轉換率 |
| 技能演化 | 需求 vs 供給落差 | 需求＝人類任務 σ 中位；供給＝worker a 中位；另拆「未填補需求」與「閒置人力」 |

> 設計提醒：「職業」＝σ 區間、「技能」＝能力 a——這是 v4 把多維收成單純量後的**投影**，不是另外加回多維度。

## 6. 分階段加回饋（不是刪，是延後——一次一個）

最小核心刻意近乎比較靜態（F 外生上爬、其餘重新排序），它**已能生成體制**。確認骨架會呼吸後，一次加一個，看清楚各自做什麼：

1. **內生前沿**（learning-by-doing）：被部署越多的 σ 區間，F 在那爬越快 → 臨界點、路徑依賴。
2. **擁擠效應**：高 σ 端湧入越多人，V 溢酬遞減 → 分配分岔（集中 ↔ 壓縮的拔河）。
3. **需求迴路**：就業/所得 → 任務生成 c → 連鎖崩潰或良性成長。
4. **適應性預期（時滯）** / **企業進出**：蛛網震盪、策略選擇。

三個一起上，永遠分不清是誰造成的——那正是 v3 的病。v4 逐一引入並各自留下相圖。

> **目前進度**：人類適應（在崗者學習 `g` + 失業者再訓練 `retrain_rate`）已實作，並產生宏觀相圖與兩張過程圖。**尚未**加入：內生前沿（learning-by-doing）、擁擠效應、需求迴路、企業進出。

## 8. v3 → v4 改動速查

| 主題 | v3 | v4 | 為什麼 |
|---|---|---|---|
| 任務 | 四維 + φ + p_true | **單一 σ** | 動態只需「離前沿多遠」 |
| 人 | structuring/verify/breadth/adapt/5技能 | **單一 a** | 其餘不承重且污染分析 |
| AI | maturity/cost/reliability/scope | **單一 F(t)** | 都只是推前沿 |
| 替代/協作 | 三通道 softmax | **σ vs F 一條門檻推導** | 一個量推出一個區分 |
| 增益 | O-ring + 驗證 + 管理幅度 | **刪除；集中改由排序匹配浮現** | 不再偷渡凸度，變可證偽 |
| 凸度來源 | 生產結構假設 | **V(σ)=σ^k，k 可掃描** | 攤開來測，不預設 |
| 新任務 | 用講的 | **生成速度 c（分配側旋鈕）** | 機制化；但宏觀體制實為 r vs 學習 g（§0.5） |
| 參數 | 8+ | **約 4** | 可識別、組成 2×2 相圖 |
| 回饋 | 全上 | **分階段一次一個** | 看得清誰造成什麼 |
| 定位 | 含混 | **探索性，先骨架後校準** | 方法論誠實 |

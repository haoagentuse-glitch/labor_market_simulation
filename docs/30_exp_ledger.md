## 歸檔規範（怎麼存，才對得起實驗）

每一輪結果都凍結在一個**版本化、永不覆蓋**的資料夾：

```
results/<版本>/
├── figures/      *.png          ← git 忽略（可由腳本重生）
├── data/         *.csv          ← 每張圖的文字版陣列，進版控、可 diff
├── summary.json                 ← 那一版所有圖的純量結論（不必看圖就能讀）
└── manifest.json                ← 參數 / 種子 / git hash / 時間（可重現）
```

- **版本字串單一來源**：`src/labor_sim/output.py` 的 `_DEFAULT_VERSION`，或執行時用環境變數
  `LSIM_VERSION` 覆蓋（例：`LSIM_VERSION=L_0.2.1 uv run python scripts/sweep.py`）。
- **圖必有對應 CSV + 一行 summary**：腳本每產一張圖，同步寫 `data/*.csv` 與 `summary.json`，
  讓之後讀結論走文字、不走像素。
- **不可重現的不算數**：每支腳本跑完都寫 `manifest.json`，記下它用的參數與種子。
- 版本語意：`L_<major>.<minor>.<patch>`。加機制或改結論 → minor；同設定重跑或修圖 → patch。

## L_0.2.1 — 校準地基（消除量測與機制的人為假象）

**狀態**：程式已就緒，待執行（`scripts/run_v021.py` → `results/L_0.2.1/`）。
**動機**：L_0.2.0 檢討（§六）指出兩個地基問題會汙染任何後續結論——
(1) t=0 就有約 25% **槽位假性失業**（人類任務數<工人數，與 AI 無關）；
(2) 學習是**無上限複利** `a*(1+g)`，20 年 ×2.6–4.2 倍，高估長期適應。
**注意**：兩個修正都以**新參數、預設關閉**實作，L_0.2.0 完全可重現（不違反「一次一個、可對照」）。

改了什麼：

1. **槽位校準**（`model.py` 加 `tasks_per_worker`）：設定→反推 `task_density` 使 t=0 人類任務數
   = ratio×工人數。實測 t=0 失業 0.254（未校準）→ 0.160（ratio=1.0）→ 0.085（ratio=1.2）；
   殘留為**匹配摩擦**（a≥σ 限制，屬合理），非槽位短缺。
2. **飽和學習**（`model.py` 加 `ability_ceiling`）：學習改 `a += g·(ceiling−a)`，趨近天花板、永不超過。
   實測末段最大能力：飽和 2.84（≤ceiling=3.0）vs 無上限 9.73。
3. **校準基線腳本** `scripts/run_v021.py`：三情境用 `tasks_per_worker=1.0, ability_ceiling=3.0`，
   並列印校準前/後對照，輸出 `results/L_0.2.1/`。

**已驗證（結果回填，見 `results/L_0.2.1/summary.json`）**：

- [x] 校準後三情境 regime **完全不變**：collapse / reallocation / reallocation（與 L_0.2.0 同）→ 方向穩健。
- [x] t=0 就業 0.84（失業 0.16），較未校準的 0.25 大幅下降；殘留為**匹配摩擦**（a≥σ），非槽位短缺。
- [x] 飽和學習（ceiling=3.0）下**重配置仍存活**（末段就業 0.973），學習即使有上限仍能救（g=0.006）。

**Headline 結果**：**L_0.2.0 的三情境結論不依賴那個約 25% 的假性失業地板**。校準後：
(1) regime 全不變；(2) **在職者 Gini 幾乎不動**（collapse 0.10 / tug 0.13 / realloc 0.19，與 L_0.2.0 一致，
且維持 collapse<tug<realloc 的排序）→ 集中度結論穩健；(3) 就業絕對水準上升（槽位校準給更多人類任務，
效果蓋過飽和學習的抑制）→ 印證「絕對就業數字是校準產物、只有方向可讀」。全體 Gini 隨失業下降而降
（realloc 0.37→0.21），再次佐證它受就業汙染、在職者 Gini 才是乾淨集中度。
**待補**：對 `ability_ceiling` 做敏感度（本輪只測 3.0；更低天花板可能讓 realloc 臨界前移）。
**產物**：`results/L_0.2.1/`

**狀態**：✅ 已執行（git b91baff），結果見 `results/L_0.2.0/`、檢討 `docs/30_exp_L0.2.0.md`、白話 `_private_notes/L_0.2.0_白話解釋.md`。

先把量尺校準、把過程現象用多種子確認、把「前沿結構性優勢」量化成一條線。
**注意**：本版三項都是**量測 / 診斷修正，不是新的模型回饋**，因此一次做完不違反
「一次只加一個回饋」的 guardrail。σ/a/F 尺度校準未動，與 L_0.1.0 可對照。

2. **B-2 過程圖多種子**（`process.py`）：單一 seed → `SEEDS=16`，畫**中位數 + 10–90 分位帶**，

基礎建設：

- [x] 在職者 Gini 隨 `ability_sigma`：**+0.128（重尾→更集中）**，與舊 top10 的 −0.583 相反

**Headline 結果**：L_0.1.0 四條結論裡，第 4 條（集中度反直覺）**推翻為量測假象**；
第 1 條（前沿主宰）**修正**為「學習有用但有硬上限」；第 2、3 條（替代≠就業、掏空升級失業同生）

## L_0.1.0 — 第一輪基線（回顧）

**產物**：`results/L_0.1.0/figures/`（前置儀器化之前，無文字版 data）。
**檢討**：`docs/30_exp_L0.1.0.md`。

四個核心結論：

1. 前沿速度不對稱主宰命運：崩潰邊界接近一條近垂直臨界線（r≈0.004），非 r=g 對角線。
2. 任務生成 ≠ 重新就業：新工作生成率遠超替代率，失業仍從 25%→42%（新工作都在高端）。
3. 掏空＋升級＋失業同時發生，由「技能能否跟上前沿」主軸驅動。
4. **集中度指標被就業水準汙染**：分配相圖反直覺顯示「越均質越集中」，疑為「倖存者稀少」假象。
   → 這正是 L_0.2.0 要校準的對象。

## 1. 問題:單調結局的質疑

**提問**:不管參數從簡單調到複雜,模擬終局都是「AI 逐漸全面取代」——這思路是不是根本錯了?

**診斷**:對 concept v1–v3 而言可以從方程式直接證明終局被寫死,來源是四個結構性元素:

1. AI maturity 單調遞增、漸近線 1(learning-by-doing 只有正回饋);
2. AI cost 單調遞減趨 0 → `value_if_AI → ∞`,任何比較最終選 AI;
3. 人類的動態保護項 `structuring_scarcity·(1−maturity)` 隨 maturity→1 消失,只剩靜態的 embeddedness;
4. 任務空間封閉(固定任務類型與四維座標)。

推論:**可數位化且低情境依附的任務被完全取代是定理,不是發現;高情境依附任務永不被取代,也是定理。**
相圖畫的是「通往同一吸引子的不同路徑」,不是不同終點。

**反向修正**:「全面取代根本不可能」同樣不可以當前提。正確目標是讓**終局成為依變項**——
模型要能在某些參數區跑出全面取代、在另一些跑出停滯或人類溢價經濟。

**缺失機制清單**(真實經濟裡擋住「全面取代」的力量):

| 機制 | 一句話 | 文獻錨 |
|---|---|---|
| AI 端負回饋(供給約束) | 算力/能源/推論容量稀缺 → AI 成本不單調趨零 → 比較利益保護復活 | Ricardo;AGI 經濟學 |
| 部門需求再配置 | AI 品趨近免費 → 支出份額流向人類部門(替代彈性<1 時) | Baumol (1967) |
| 開放任務空間 | 自動化 vs 新任務創造的賽跑,勝負不該預設 | Acemoglu & Restrepo (2018) |
| 工資作為均衡變數 | 取代要求 AI 成本 < 人類保留工資;核心指標應含**勞動所得份額** | 一般均衡 |
| 能力天花板不確定 | 「AI 終將會做一切」是編碼進函數形式的信念;天花板應為隨機變數,由部署揭露 | Knightian 不確定性 |

## 2. 脈絡(一):歷史作為校準點(不是類比)

每次技術革命 = 同一模型在參數空間的**已知座標點**;相圖若能把歷史片段放進正確區域,AI 的問題就從「預測」變成「定位」:

- **農業機械化**:調整靠**世代交替**(農夫的兒子不務農),不是再訓練 → 模型需要世代結構。
- **Engels' Pause(1790–1840)**:產出漲、實質工資停滯近半世紀 → 「工資份額脫鉤一整代」是真實體制;
  過渡期可長於一個工作生涯,transient 本身就是福利問題。
- **電氣化(David 1990)**:紅利遲到四十年,卡在互補組織資本 → 採用延遲應內生,不是常數 delay。
- **ATM/織布工(Bessen)**:需求彈性高時自動化**增加**就業,飽和後才摧毀 → 彈性應隨飽和遞減,
  內生產生「就業倒 U」。
- **馬(唯一的全面取代實例)**:三條件同時成立才發生——無所有權、無需求角色、工資降無可降。
  → 「人類會不會變成馬」由 **AI 資本所有權分佈** 這個狀態變數決定,可畫成相圖上一條線。
- **校準錨**:今日約六成職業 1940 年不存在(reinstatement 速率的歷史基準)。

## 3. 脈絡(二):螺旋史觀——不變量與旋轉律

假設歷史是向上螺旋(結構重演、維度旋轉),歸納出:

**五個不變量**:
1. 溢酬永遠流向「新便宜品的互補品」(預測工具:問 AI 把什麼變便宜,而非人類還剩什麼);
2. **避難所自我掏空**:把技能標準化/教材化以規模化 = 為自動化準備語料(LLM 正是吃顯性化語料長大);
3. 每圈誕生的「中介職業」(操作工→打字員→程式設計師→prompt engineer)都是過渡性的;
4. 每圈都有「這次沒地方逃」文獻——全錯在終點、全對在過渡;N 次平安不能歸納出第 N+1 次(貝葉斯更新,到不了 1);
5. 圈周期在縮短;痛苦由「圈周期/世代長度」比值決定。

**旋轉律**:稀缺軸的軌跡(力量→操作→例行認知→分析→結構化)一直在**能力軸**上;下一圈可能整個轉出能力軸——
轉向**責任**(可究責性)、**偏好**(知道什麼值得要)、**真人性**(human-made 本身是產品)、**所有權**(對機器產出的請求權)。
馬被完全取代,正因馬只活在能力軸上。

**問題的最終形態**:不是「AI 會不會全面取代」(能力軸上的偽問題),而是
「**螺旋下一圈會不會轉出能力軸;轉出去之後,新軸還載不載得動大眾就業**」——答案由需求端與所有權分配決定,不由 AI 能力決定。

## 5. 產出:候選機制清單(供 concept v5 / spec 迭代取用)

依「一次只加一個回饋」憲法排序建議:

1. **F 的天花板/減速**(AI 端負回饋)——最小改動:`F' = r·(F_ceiling − F)`,F_ceiling 可掃描或設為隨機變數 → 直接檢驗「終局是否仍單調」。
2. **需求迴路**(spec 既定計畫)+ 勞動份額指標。
3. **世代結構** + 主比值「圈周期/世代長度」進 SWEEP。
4. **編碼化回饋** vs 內生前沿的對照實驗。
5. **維度出生**(λν 本體論就緒後):新維度以一定速率誕生、與當前 F 正交——螺旋旋轉的機制化。
6. 驗證目標新增:模型是否自發跑出「中介職業興衰弧線」(協作帶份額先升後降)。

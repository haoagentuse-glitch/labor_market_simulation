## 1. 系統一頁

| 實體 | 純量 | 意義 |
|---|---|---|
| 任務 | `σ` | 離 AI 前沿多遠 = 需要多少「賦予結構」的能力 |
| 人 | `a` | 能力,LogNormal 異質分佈 |
| AI | `F(t)` | 前沿,以速度 `r` 線性上爬 |

```
任務供給:σ ~ U[0, σ_max(t)],  σ_max(t) = σ_max0 + c·t     # c = 邊緣新任務生成速度
規則(每步每任務):
  σ < F − ε   → AI 獨力完成(替代),人不在場          # ε = 前沿毛邊雜訊
  σ ≥ F       → 需 a ≥ σ 的人完成,報酬 V(σ) = σ^k
匹配:合格者(a ≥ F)排序匹配——σ 由大到小、a 由大到小,能做就配(貪婪 O(n log n))
適應:在崗者 a *= (1+g);失業者 a *= (1+retrain_rate)
     (地基選項:飽和式 a += g·(ability_ceiling − a),見 §3.4)
a 持續 < F 者配不到任何任務 → 失業
```

這條規則**免費生成**(皆非設定):替代/協作兩區(一條門檻切出)、存活門檻(= 當下的 F)、逃跑賽跑(r vs g)、集中/壓縮(排序 × 能力尾巴 × V 凸度的後果,可被 k=1 或平尾推翻——這是 concept v3 的 O-ring 做不到的可證偽性)。

## 3. 機制註記

### 3.1 設計修正紀錄:宏觀賽跑是 r vs g,不是 r vs c

原設計以為「前沿 r vs 任務生成 c」決定崩潰/重配置。第一版程式**否證**了它:c 只在前沿之上加任務,而那些任務只有高能力者能接——c 抬高頂端機會與集中度,救不了被替代的中低能力者;靜態能力下純核心只會崩潰。真正切換體制的是 **r vs g**:r≫g 在崗者也被追上(崩潰);g≳r 在崗者守住身位(重配置)。故宏觀相圖掃 `(r,g)`,c 降為分配側旋鈕。(這正是最小模型的價值:否證 naive 設計;過程詳見 `50_sparks` 第 4 則與 `30_exp_L0.1.0`。)

### 3.2 毛邊前沿 ε

可替代性**就是** σ 對 F 的比較,不另立 p_true;ε 讓門檻帶噪音,即毛邊前沿的最小形式。

### 3.3 排序匹配為何是最小且不偷渡

集中若出現,必須是「排序 + 重尾 + 凸 V」的後果:調平尾巴(`ability_sigma`→0)或調 k=1 後仍集中,即機制錯——攤開來測,不預設。

### 3.4 地基選項(L_0.2.1,預設關、向後相容)

`tasks_per_worker` 反推 task_density 消除 t=0 槽位假性失業(0.254→0.160@ratio=1.0);`ability_ceiling` 把學習改飽和式,杜絕 20 年 ×2.6–4.2 的無上限複利。兩者皆為校準,不是新回饋。

## 4. 輸出指標

**結果指標**(每步):`employment_rate`、`gini`(全體)、`gini_employed`(在職者,乾淨集中度)、`top10_share`、`human_zone = max(σ_max − F, 0)`、`F`、`sigma_max`。
**過程指標**(v4.1,兩張機制圖):職業層級人口比例(σ 切層 + 失業)、AI 替代率(σ<F 價值佔比)、新職業生成率(σ>σ_max0 價值佔比)、平均技能(全體/在職者,疊 F)、再訓練成功率(失業→就業轉換率)、需求 vs 供給落差(任務 σ 中位 − 能力 a 中位)。
**體制判定**:order parameter 客觀分類(就業末值、human_zone 是否關閉、頂端份額),不靠眼睛。「職業」=σ 區間、「技能」=a——是單純量的投影,不是加回多維。

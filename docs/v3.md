# AI 時代勞動市場演化模擬：研究框架與設計文件 v3

> 修訂自 v2。v2 把「會把假設原樣吐出來的機器」改成了會湧現的交互系統；**v3 的核心貢獻是：把模型裡最關鍵的那條函式（人用 AI 後的產出），從「設定」降級為「推導的結果」**——並沿途釐清了一個 v2 沒講清的根本問題：模型裡的 AI 到底是「純替代」還是「協作」。

> v1 的文獻與四維框架（A 級）原封保留；v2 的回饋迴路設計保留；v3 重寫增益函式並補上它的微觀基礎。

## 0. 版本演進與 v3 的三個新支柱

| | v1 → v2 | v2 → v3 |
|---|---|---|
| 解決的病 | 計分函數把結論寫死、無回饋、無一般均衡 | AI 的角色混淆、增益函式是憑空設定的 |
| 關鍵動作 | 加入隨機前沿、擁擠、learning-by-doing、需求迴路 | 拆出雙通道、推導增益函式、能力重尾 |

v3 的三個新支柱：

1. **雙通道 AI（§3.4）**：把「純 AI 替代」與「人＋AI 協作」分成兩條一級通道。v2 把它們混在一起，導致「AI 是替代還是放大」這個問題無解。
2. **推導出來的增益函式（§3.5）**：人用 AI 後的產出，不再是手調的 log 或 n²，而是由三個獨立、可測量的機制推出來——O-ring 多子任務（中段凸度）、驗證門檻（底部）、管理幅度（頂部）。整條 S 曲線是後果，不是假設。
3. **能力重尾 + 分岔（§3.6）**：結構化能力是重尾分佈，擠進結構化端的人被能力天花板擋住，所以溢酬不是「崩」而是「分岔」——可能集中成超級明星，也可能被壓縮，由參數決定。

## 一、研究定位（沿用 v2）

### 核心提問

> **在 AI 前沿持續擴張下，勞動市場會浮現哪些不同的重組「體制」（regimes）——平穩重配置、極化塌陷、協作擴張？是什麼*條件*決定市場落入哪一種，這些體制之間是否存在臨界點（phase transition）？**

依變項是「市場落入哪種體制」，不是「哪種特質贏」。特質與增益函式的形狀是讓體制浮現的**機制**，不是被驗證的預設答案。

三個層次：

- **體制問題（macro，主軸）**：市場落入哪種型態？臨界點在參數空間哪裡？
- **動態問題（meso）**：AI 前沿擴張 × 人類逃逸 × 需求回饋 × 不均演化，交錯走出什麼軌跡？
- **個體問題（micro，輔助）**：哪種特質組合存活——**這是觀察結果，不是設計目標。**

### 招牌交付物：體制相圖

在「AI 成長速度 × 人類學習速度」（及其他關鍵參數對）平面上，標出每區落入哪種體制、臨界線在哪。v3 多了一個維度可掃描：**任務組合的結構化分佈**——因為它現在直接決定不均會集中還是壓縮（§3.5–3.6）。

### 為什麼選 ABM

不做經濟預測，做**體制辨識**；市場動態從 agent 交互浮現；可跑受控政策實驗並比較體制轉移。

## 三、模型設計（v3 核心重寫）

### 3.0 設計總原則

| 機制 | 回饋/性質 | 解決的病 |
|---|---|---|
| 隨機/可學習替代性（§3.3） | 不確定 + 試誤 | 套套邏輯 |
| **雙通道 AI（§3.4）** | 結構釐清 | 「替代 or 放大」無解 |
| **推導的增益函式（§3.5）** | 微觀基礎 | 函式憑空設定 |
| **能力重尾 + 分岔（§3.6）** | 負回饋 + 重尾 | 「高 structuring 永遠贏」/「溢酬一起崩」 |
| AI learning-by-doing（§3.7） | 正回饋 | 外生 S 曲線 |
| 需求面迴路（§3.8） | 乘數 | 無一般均衡 |
| 企業進出（§3.9） | 演化篩選 | 公司不會死 |
| 適應性預期 + 時滯（§3.10） | 蛛網循環 | 完美預期 |

### 3.1 世界架構

```
World
├── Agents
│   ├── Workers（人類，結構化能力重尾分佈）
│   ├── AI Capability Pool（按任務類型，能力內生成長）
│   └── Companies（會進場/破產）
├── Task Market（需求量內生；每任務有隱藏可替代性 + 結構化子任務比例 φ）
├── Product Market（價格、總需求迴路）
├── Matcher（三選一：純AI / 人+AI協作 / 純人；帶噪音試誤）
├── Belief / Expectation System（有時滯）
├── AI Frontier Engine（learning-by-doing；管理幅度上限隨之抬高）
└── Statistics & Regime Classifier
```

### 3.2 每個 timestep 流程

```
1. Product Market：上期就業/所得 → 本期總需求 → 各類任務需求量
2. Task Market：生成任務，隱藏 p_true、結構化子任務比例 φ
3. Companies 依「信念」在三通道間挑 value/cost 最高者
4. Matcher 指派；AI 成功與否抽樣（暴露真實可替代性）；協作產出依 §3.5 增益函式
5. 執行 → 產出、薪資、利潤
6. 回饋更新：AI maturity（部署驅動）、擁擠（能力相依）、信念（時滯）
7. 選擇：虧損公司破產；偶有新公司進場
8. Statistics + Regime Classifier 更新
```

> **強化版設計原則**：沒有任何一條方程式直接寫出第六節的結論；連那條增益函式都是推導來的，不是設的。

### 3.3 替代性：隨機 + 可學習（沿用 v2，最關鍵的去套套邏輯）

每個任務有**隱藏的**真實可替代性 `p_true`，公司事前不知道，只能試了才知道：

```python
p_true = clip(
    sigmoid(  w_d*digitizability
            - w_e*contextual_embeddedness
            - w_s*structuring_scarcity*(1 - ai_maturity[task_type])
            + noise(scale=sigma_frontier) ),   # 毛邊前沿
    0, 1)
# 公司只有帶誤差的 belief_p；AI 接手成功 ~ Bernoulli(p_true)；用結果貝氏更新
```

效果：前沿隨機毛邊（操作化 Dell'Acqua 的模糊性）、公司會犯錯、「結構化值錢」要由市場發現而非被保證。`sigma_frontier` 是關鍵掃描參數。

### 3.4 雙通道 AI：替代 vs 協作（**v3 新增，釐清 v2 的根本混淆**）

v2 把 AI 當「派給 AI / 派給人」二選一，這隱含 AI＝純替代，卻又在談架構師時用「人指揮 AI」的協作故事——兩者混在一起，導致「AI 是替代還是放大」無法回答。

v3 明確分成**三條通道**，公司挑「每單位價值最便宜」者：

```python
# 通道一：純 AI（替代）—— 人消失（Acemoglu-Restrepo displacement effect）
pure_ai:        output = ai_capability(task),         cost = ai_cost
# 通道二：人 + AI（協作）—— 人還在，AI 放大其產出（productivity effect）
collaboration:  output = augmented_output(s, a, task),cost = wage + ai_cost   # ← §3.5 增益函式住這
# 通道三：純人 —— 用在 AI 幫不上忙的地方
pure_human:     output = human_output(s, task),       cost = wage
```

關鍵釐清：

- 「用 AI 會不會變厲害、變多厲害」這個問題**只存在於通道二**。通道一裡人不在場，談不上誰受惠。
- 所以增益函式（§3.5）是協作通道的屬性，不是替代通道的。
- 兩通道會撞出 v3 的核心分配悖論（§六）：協作通道可能**相對**抬高某些人，替代通道同時**抽掉**另一些人的地板。

### 3.5 協作增益函式：從「設定」到「推導」（**v3 核心**）

不再手調 log 或 n²。整條曲線由三個獨立、可測量的機制推出，分別決定 S 曲線的**底、中段、頂**。

#### (a) 中段凸度 → O-ring 多子任務（推出 β = φn）

任務 = n 個子任務，**全部要對**才有價值（Kremer 1993 O-ring，生產相乘）。比例 φ 是結構化子任務（綁人，品質隨能力 s），比例 (1−φ) 是執行子任務（AI 可做，品質隨 AI 能力 a）：

```
Y = V · q_E(a)^{(1−φ)n} · q_S(s)^{φn}
令 q_S(s)=s, q_E(a)=a：
Y = V · a^{(1−φ)n} · s^{φn}
```

**對 s 的指數 β = φn 是推出來的，不是設的**：

- φ 高（結構化密集）→ β 大 → **凸、放大**（β=2 即字面的 n²）。
- φ 低（執行密集）→ β→0 → Y≈a^n，**幾乎與人能力無關 → AI 主導、壓縮**。

這一步同時推出：凸放大的存在 + **β 為何隨 structuring_scarcity 滑動**。且 φ 可由任務分解資料（如 O*NET）估計——參數可校準。

#### (b) 底部門檻 → 驗證能力（菜鳥只能當聊天機器人）

AI 輸出有錯誤率，**人得有能力判斷它對不對**才榨得出價值：

```
有效 q_E = a · P_verify(s)，   P_verify(s) → 0  當 s < s_min
```

低能力者不敢用 AI 成果（分不出對錯）→ 榨不出價值。**S 曲線平坦底部是驗證門檻推出來的。** P_verify 可用實測錯誤偵測率校準。

#### (c) 頂部飽和 → 管理幅度（終究有極限）

一個人注意力固定，能同時監督的子任務數有上限（Lucas 1978 span-of-control）：

```
n(s,a) ≤ T / τ(a)        # 總注意力 T / 每子任務監督成本 τ
```

AI 再強也只能盯這麼多。**S 曲線的頂是注意力預算推出來的；** AI 變強降低 τ → 上限隨時間抬高（接回前沿演化）。

#### 合成：整條增益函式

```python
def augmented_output(s, a, task):
    phi   = task.structuring_share              # O-ring 結構化子任務比例
    n     = min(task.natural_subtasks, T / tau(a))   # 頂：管理幅度
    qE    = a * P_verify(s)                     # 底：驗證門檻
    qS    = s
    return V * qE**((1-phi)*n) * qS**(phi*n)    # 中：O-ring，β=phi*n
```

每個零件掛在可測的量上：φ（任務分解）、P_verify（錯誤偵測率）、T/τ（一個人實際能管幾個 AI agent/工具）。**不再調曲率，而是校準機制。** 若資料顯示曲線長別樣 → 知道哪個機制假設錯了。

> 誠實界線：仍要選 q_E、q_S 形式與少數常數；推導把「整條形狀」從自由度移除，剩下少數有意義、可校準的參數。最佳實務：**推導骨架，只對殘餘參數做敏感度掃描。**

### 3.6 能力重尾 + 擁擠分岔（**修正 v2「溢酬一起崩」**）

v2 假設人人可湧入結構化端、溢酬一起崩——錯在假設能力均質。v3：

```python
s_i ~ LogNormal(mu, sigma)     # 結構化能力重尾：少數人能當架構師（sigma 控不均度）
# 擁擠只作用於「真正能勝任的有效供給」，不是全部湧入者
effective_supply = count(workers where augmented_output(s_i, a, task) > task_bar)
premium = base_premium / (1 + gamma * effective_supply)
```

因為 `effective_supply` 被重尾天花板鎖死，前沿外（高 φ）任務的溢酬**黏著、集中**而非崩潰。於是：

- 高 φ 任務（β>1，放大）+ 重尾能力 → **集中成超級明星**（Kremer O-ring 複利、Rosen 1981 superstars）。
- 低 φ 任務（β<1，壓縮）→ AI 把菜鳥拉到平均（Brynjolfsson 客服研究）。

**最終市場集中還是壓縮，由任務組合 × 移動前沿 × 三股力量（重尾 σ、凸度 φn、替代地板移除速度）的拔河決定——是湧現，不是預設。** 三股力量都要進 SWEEP，任一調死都會洗平動態。

### 3.7 AI Frontier：learning-by-doing（沿用 v2）

ai_maturity[t] += alpha * deployment_volume[t] * (1 - ai_maturity[t])  # 部署驅動成熟（正回饋）

## 四、研究假設與情境（可證偽 + 體制掃描）

### 4.1 可證偽假設（每條附「什麼觀察會推翻它」）

- **H1（中段塌陷）**：AI 降本 > 人類學習時，「高數位化 × 低結構化」中段最先崩。*證偽*：若該區中段就業率不顯著低於兩端。
- **H2（結構化溢酬非單調）**：高 structuring worker 溢酬先升；但**擁擠 + 能力重尾**決定它最終是集中還崩。*證偽*：若不論 σ、φ 如何，溢酬都單調 → §3.5–3.6 機制無效。
- **H3（政策只延後不改變）**：再教育只延遲失業波。*證偽*：若需求迴路讓補貼撐住所得→撐住需求→改變最終分佈。
- **H4（路徑依賴）**：learning-by-doing 讓早期隨機選擇鎖定長期結構。*證偽*：不同種子收斂同一結構。
- **H5（分配悖論，v3 新增）**：協作通道壓縮在崗者差距，替代通道同時抽掉低能力者地板——**淨不均方向由任務的 φ 分佈決定**。*證偽*：若改變 φ 分佈不改變不均走向 → 雙通道機制無作用。

### 4.2 三種情境（結果欄＝待相圖判定）

| 情境 | AI 成長 | 人類學習 | 態度 |
|---|---|---|---|
| 慢替代 | 低 | 中 | 待相圖 |
| 快替代 | 高 | 低 | 可能因需求崩潰更糟，或新任務緩衝更輕 |
| 協作型 | 高 | 高 | 協作紅利可能被擁擠/集中稀釋 |

v3 新增第三條掃描軸：**任務的結構化比例 φ 分佈**（執行密集 ↔ 結構化密集的經濟）。

### 4.3 產業代表性（沿用 v1 四類）

實體服務、知識密集、創意、軟體開發——四維座標沿用 v1。

- **新輸出**：技能別薪資曲線、Gini、structuring 溢酬曲線、**不均集中度指標**。

### Phase 3：擁擠分岔 + AI 內生演化 + 需求迴路（約 2 週）
- 能力相依擁擠效應上線（看溢酬集中 vs 壓縮）。
- learning-by-doing + 規模降本 + 管理幅度上限抬高。
- Product Market 需求迴路。
- **這時三條曲線交錯 + 回饋 + 分配悖論才會全部跑出來。**

### Phase 4：企業進出 + 政策 + 相圖（約 1–2 週）
- 公司進出（策略被選擇）。
- 政策 A/B（UBI、再教育、AI 稅）比較體制轉移。
- 掃描參數平面（含 φ 分佈軸），產生體制相圖。

## 六、預期觀察到的社會現象（候選浮現型態，需模擬證實/推翻）

1. **中段塌陷**——可能；但需求迴路或大前沿噪音下未必最慘。
2. **結構化溢酬的分岔（v3 升級）**——高 φ 任務 + 重尾能力 → 集中成超級明星；低 φ 任務 → 壓縮。最終看任務組合，非單調。
3. **過渡性失業波 + 二階連鎖**——需求迴路下可能放大成連鎖崩潰，或被新任務平滑。
4. **新職業浮現**——要求機制（Acemoglu-Restrepo reinstatement）：新任務從前沿釋放的人力 + 需求內生產生。
5. **路徑依賴與不可逆**——learning-by-doing 鎖定早期隨機選擇。
6. **體制間臨界轉移**——相圖相變線：參數微變使市場從「協作擴張」翻入「極化塌陷」。
7. **分配悖論（v3 核心新增）**：

   > 協作通道裡 AI **相對**抬高低能力在崗者（壓縮）；替代通道同時**抽掉**執行類工作的地板。「AI 讓弱者在還活著的崗位上相對變強」與「AI 讓弱者根本沒崗位可活」同時成立——淨效果由經濟的 φ 分佈與前沿位置決定，隨時間呼吸。

## 附錄：v2 → v3 改動速查

| 主題 | v2 | v3 | 為什麼 |
|---|---|---|---|
| AI 的角色 | 替代/協作混在一起 | **三通道明分**（純AI/協作/純人） | 「替代 or 放大」才有解 |
| 人用 AI 的增益 | 手調 log / n²（設定） | **三機制推導的 S 曲線**（O-ring+驗證+管理幅度） | 從「設定」變「後果」 |
| 凸度從哪來 | 假設 | **β = φn，由 O-ring 推出**，隨 structuring_scarcity 滑動 | 可校準、有根據 |
| 能力分佈 | 隱含均質 | **重尾 LogNormal** | 「擠進來」被天花板擋住 |
| 擁擠結果 | 溢酬一起崩 | **分岔**（集中 superstar vs 壓縮） | 更真實、更湧現 |
| 不均 | 較少著墨 | **分配悖論**：協作壓縮 + 替代抽地板 | v3 核心發現點 |
| 掃描軸 | 兩條曲線速度 | + **能力重尾度 σ + 任務 φ 分佈** | 集中/壓縮的關鍵旋鈕 |
| 新文獻 | — | Kremer、Lucas、Rosen、Brynjolfsson | 增益函式的微觀基礎 |

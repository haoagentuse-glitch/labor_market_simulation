## 歸檔規範（怎麼存，才對得起實驗）

每一輪結果都凍結在一個**版本化、永不覆蓋**的資料夾：

```
results/<版本>/
├── figures/      *.png          ← git 忽略（可由腳本重生）
├── data/         *.csv          ← 每張圖的文字版陣列，進版控、可 diff
├── summary.json                 ← 那一版所有圖的純量結論（不必看圖就能讀）
└── manifest.json                ← 參數 / 種子 / git hash / 時間（可重現）
```

- **版本字串單一來源**：`src/labor_sim/output.py` 的 `_DEFAULT_VERSION`，或執行時用環境變數
  `LSIM_VERSION` 覆蓋（例：`LSIM_VERSION=L_0.2.1 uv run python scripts/sweep.py`）。
- **圖必有對應 CSV + 一行 summary**：腳本每產一張圖，同步寫 `data/*.csv` 與 `summary.json`，
  讓之後讀結論走文字、不走像素。
- **不可重現的不算數**：每支腳本跑完都寫 `manifest.json`，記下它用的參數與種子。
- 版本語意：`L_<major>.<minor>.<patch>`。加機制或改結論 → minor；同設定重跑或修圖 → patch。

## L_0.2.0 — 校準量尺（B 清單：先把「量到的東西」弄準）

**狀態**：✅ 已執行（git b91baff），結果見 `results/L_0.2.0/`、檢討 `docs/RESEARCH_REVIEW_v0.2.0.md`、白話 `_private_notes/L_0.2.0_白話解釋.md`。
**動機**：L_0.1.0 有一個結論被懷疑是量測假象（集中度被就業水準汙染）。在加任何新機制前，
先把量尺校準、把過程現象用多種子確認、把「前沿結構性優勢」量化成一條線。
**注意**：本版三項都是**量測 / 診斷修正，不是新的模型回饋**，因此一次做完不違反
「一次只加一個回饋」的 guardrail。σ/a/F 尺度校準未動，與 L_0.1.0 可對照。

改了什麼：

1. **B-1 拆開集中度指標**（`model.py`）：`Result` 新增 `gini_employed`（只算 `earnings>0`），
   與舊的全體 `gini`、`top10_share` 並存（加欄位，向後相容）。`sweep.py` 的分配相圖改成
   **top10 份額 vs 在職者 Gini 並列**，直接驗證/推翻 L_0.1.0「能力越均質越集中」。
2. **B-2 過程圖多種子**（`process.py`）：單一 seed → `SEEDS=16`，畫**中位數 + 10–90 分位帶**，
   確認掏空 / 升級 / 錯配不是特例。
3. **B-3 量化臨界前沿速度 r\*(g)**（`sweep.py`）：由就業率網格內插出每個 g 的崩潰臨界 r，
   新出 `critical_frontier.png` 與 `r*(g)` 斜率（斜率≈0 → 前沿主宰；斜率大 → 學習能救）。
4. **B-4 最小敏感度**（`sensitivity.py`，可選）：對 `F0` / `ability_median` / `task_density`
   各 ±20%，檢查三情境 regime 是否翻面。為此 `Params` 加 `ability_median`（預設 0.5，不改預設行為）。

基礎建設：

- 新增 `src/labor_sim/output.py`：版本化路徑、CSV / `summary.json` / `manifest.json` 寫入。
- 目錄重構：舊 `figures/L_0.1.0/` → `results/L_0.1.0/figures/`；新結果一律進 `results/<版本>/`。
- `.gitignore`：`results/**/figures/` 忽略，`data/` 與兩個 json 進版控。

**已驗證（結果回填）**：

- [x] 在職者 Gini 隨 `ability_sigma`：**+0.128（重尾→更集中）**，與舊 top10 的 −0.583 相反
      → L_0.1.0「越均質越集中」**推翻**，確認為失業汙染假象。
- [x] `r*(g)` 斜率 **0.425**，低 g 區有斜率、約 r≈0.0043 **封頂** → 非純垂直；學習有用但有上限。

- [x] ±20% 尺度擾動 **0 次 regime 翻面** → 結論對尺度穩健。

**Headline 結果**：L_0.1.0 四條結論裡，第 4 條（集中度反直覺）**推翻為量測假象**；
第 1 條（前沿主宰）**修正**為「學習有用但有硬上限」；第 2、3 條（替代≠就業、掏空升級失業同生）
**強化確認**且更極端。量尺校準完成，可進 A 清單（建議序：內生前沿 → 需求面 → 分租）。

## L_0.1.0 — 第一輪基線（回顧）

**產物**：`results/L_0.1.0/figures/`（前置儀器化之前，無文字版 data）。
**檢討**：`docs/RESEARCH_REVIEW_v0.1.0.md`。

四個核心結論：

1. 前沿速度不對稱主宰命運：崩潰邊界接近一條近垂直臨界線（r≈0.004），非 r=g 對角線。
2. 任務生成 ≠ 重新就業：新工作生成率遠超替代率，失業仍從 25%→42%（新工作都在高端）。
3. 掏空＋升級＋失業同時發生，由「技能能否跟上前沿」主軸驅動。
4. **集中度指標被就業水準汙染**：分配相圖反直覺顯示「越均質越集中」，疑為「倖存者稀少」假象。
   → 這正是 L_0.2.0 要校準的對象。
